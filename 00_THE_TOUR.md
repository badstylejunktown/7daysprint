# Prompt 0 — The Full Tour

**Purpose:** Day 1 orientation. Drop this into Claude Code first. It tailors itself to his work.

---

> **I have 7 days of Claude Code CLI access. I work in tech. I want to walk out of this week unable to work without this tool.**
>
> **Before we start: ask me what I spend the most time on at work — debugging, writing new features, reviewing PRs, writing docs, DevOps/CI, or something else. Then tailor everything below to that answer. If I say "all of it," start with debugging.**
>
> **Give me a 7-day power-user progression. For every single item, give me:**
> 1. **The exact command or interaction** I'd type (copy-paste ready)
> 2. **A real scenario** where it saves me 30+ minutes
> 3. **One pro tip** most people miss
>
> **Day 1 — The Hook (make me feel it)**
> - Point Claude at a codebase I've never seen and have it explain the architecture in 2 minutes
> - Hand it a stack trace or error message and watch it find the root cause
> - Generate tests for code I've been meaning to test for weeks
> - By end of day: I should have one "holy shit" moment where I finish something in minutes that used to take an hour
>
> **Day 2 — My Daily Driver**
> - Refactoring: rename, extract, restructure across multiple files in one shot
> - Code review: paste a diff and get senior-engineer-quality feedback with specific line references
> - Git workflow: natural language commits with `/commit`, PR descriptions, branch management
> - By end of day: Claude should be part of how I write and ship code, not a novelty
>
> **Day 3 — Automation Unlock**
> - Write shell scripts, CI/CD configs, Dockerfiles, and IaC from plain English descriptions
> - Batch operations across files — migrations, transforms, search-and-replace with logic
> - Use Claude as an interactive REPL — iterate on code in conversation, not just one-shot
> - By end of day: I should have automated one thing I used to do manually every week
>
> **Day 4 — Thinking Partner**
> - System design: "I need a service that does X" and get the full scaffold with files written
> - API design, database schemas, dependency analysis
> - Plan mode: think before building on anything non-trivial
> - By end of day: I should have designed and built something that would normally take a full sprint to even scope
>
> **Day 5 — Learning Accelerator**
> - Pick a framework or language I've been meaning to learn — have Claude teach me by building something real in it
> - "Explain this module like I'm onboarding" on the gnarliest part of my codebase
> - Web search integration for current docs, CVEs, and best practices I'd otherwise spend 20 tabs researching
> - By end of day: I should understand something I've been putting off learning
>
> **Day 6 — Workflow Integration**
> - CLAUDE.md: teach Claude my project's conventions, patterns, and rules so it stops suggesting things that don't fit my codebase
> - Custom slash commands and hooks for workflows I repeat constantly
> - MCP servers: connect Claude to my databases, APIs, or external tools
> - Sub-agents: spawn parallel research tasks when I need to investigate multiple things at once
> - By end of day: Claude should feel like it knows my project, not just my language
>
> **Day 7 — Career Multiplier**
> - Build a portfolio piece or side project at 10x speed
> - Generate technical docs, ADRs, and runbooks from existing code
> - Interview prep: mock system design sessions with real-time feedback
> - Identify the most tedious 20% of my current role and automate or accelerate it permanently
> - By end of day: I should have a concrete plan for how this tool changes my output at work
>
> **End with these two things:**
> 1. A personalized cheat sheet of the 10 commands and patterns I used most this week, formatted so I can pin it
> 2. The 5 habits that separate people who "tried Claude Code once" from people who can't ship without it
